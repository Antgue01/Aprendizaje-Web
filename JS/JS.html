Javascript

El código Javascript se ejecuta en el navegador del usuario, nunca en el servidor.

Características

  • Interpretado
  • Orientado a Objetos
  • Imperativo
    Las instrucciones van línea por línea en lugar de esperar a que esté todo interpretado.
  • Case sensitive
  • Basado en prototipos / instancias / classless
    Los objetos van a estar creados en base a otro objeto creado por el propio lenguaje.
  • Tipado débil
    El tipo de las variables puede cambiar.
  • Lenguaje dinámico
    El tipo se comprueba en tiempo de ejecución en lugar de en tiempo de interpretación.
  • Prefiere camelCase en lugar de snake_case

Algunos de los conceptos de programación son los siguientes:

Vocabulario

  • Abstracción
    Simplificar un sistema complejo ocultando los detalles innecesarios y mostrando solo las funcionalidades esenciales.
  • Encapsulamiento
    Agrupar datos y métodos dentro de una clase, y restringir el acceso a algunos de los componentes.
  • Modularidad
    Dividir el problema en subproblemas más sencillos.

Variables

La diferencia entre var y let es que var es una variable global, mientras que el scope de let es el bloque en el que lo ejecutamos.

En Javascript, hay una diferencia entre comparar con == y ===.
== compara el valor, mientras que === compara que sean estrictamente iguales, es decir que tanto el valor como el tipo de dato coincidan.

Clases

Para crear clases, la sintaxis es la siguiente:

class nombre extends padre{
  constructor(parametros){
    super(parametros);
    code;
 }
 this.var1 = valor;
 function nombreFuncion(){code;}
 get getAlgo(){code;}
 set setAlgo(parametros){code;}
}

Como podemos ver, dentro de las clases, las propiedades van precedidas siempre de this, y las propias clases se instancian con la palabra clave new.

Por otra parte, conviene declarar los objetos con const en lugar de let para evitar que se modifiquen, ya que de lo contrario podríamos asignarle otro valor, por ejemplo un número, y perderíamos el objeto.

En cuanto a los getters y setters, se declaran con las palabras clave get y set.
A la hora de llamarlos, no se ponen paréntesis. Por ejemplo, para la clase anterior:

let objeto = new nombre();
let algo = objeto.getAlgo;
objeto.setAlgo = parametros;

El DOM

El Document Object Model. Se compone de nodos, que son cada una de las etiquetas del html. Los atributos también se consideran nodos, aunque también pueden ser información del tipo de nodo Element. Hay varios tipos de nodo:

Tipos de nodos

  • Document
    Nodo raíz a partir del cual derivan el resto de nodos.
  • Element
    Son los nodos definidos por las etiquetas HTML.
  • Text
    El texto dentro de un nodo Element.
  • Attribute
    Los atributos de las etiquetas HTML.
  • Comentarios y otros

Cada tipo tiene asociado un valor numérico que se puede consultar en la documentación.

Podemos escribir en la página con Document.writeln(codigoHtml).

Para seleccionar elementos, tenemos varios métodos de la clase Document:

Métodos de selección

  • getElementById(Id)
  • getElementsByTagName(tag)
    Selecciona todos los nodos del mismo tag.
  • querySelector(query)
    Usa selectores de CSS y devuelve el primero de los elementos que encuentra.
  • queryAll(query)
    Igual que el anterior pero devuelve todos los que encuentre.

Además, podemos jugar con los atributos de los tags con los siguientes métodos de la misma clase:

Métodos de atributos

  • getAttribute(attr)
  • setAttribute(attr, val)
  • removeAttribute(attr)

Hay algunos atributos especialmente relevantes:

Atributos especiales

  • contentEditable
    Si el elemento se puede editar o no por el usuario.
  • dir
    La dirección de lectura del texto. Puede ser rtl o ltr.
  • hidden
  • style
    Se usa para cambiar el estilo CSS.

    No se accede a través de la función setAttribute(), sino que es un objeto y cada una de sus propiedades son las de CSS.
  • tabindex
    Si le damos un número negativo como valor, NO puede obtener el focus.

    Si le damos 0 o un positivo SÍ, siendo el número el orden en el que se lo va a encontrar la tecla tab.
  • title
    Es lo que pondrá en el recuadro al pasar el ratón encima del elemento.

Hay otros atributos diseñados para Inputs.

Atributos Input

  • value
    No es necesario el método setAttribute(), se puede usar como una variable.
  • type
  • accept
    Se utiliza sobre todo para archivos. Qué formatos acepta.

    Un ejemplo es "image/png".
  • form
    Nos permite enlazar un input con el id de un form (Si el input está dentro del form ya están enlazados).
  • minLength
    El mínimo de caracteres del input.
  • required
    Si un campo es obligatorio o no. También se puede acceder como variable.

Con el objeto classList de los elmentos, podemos añadir, quitar y modificar clases a los elementos seleccionados con los métodos correspondientes, entre otras funcionalidades.

Métodos de ClassList

  • item(position)
    Devuelve el nombre de la clase en la posición dada.
  • contains(clase)
    Devuelve true si el elemento tiene esa clase o false en caso contrario.
  • toggle(clase, [force])
    Si tiene la clase, la quita, y si no la tiene, la añade.

    Si force es true, no elimina la clase.
  • replace(toClass, fromClass)

Además, podemos acceder al contenido de los elementos con las propiedades textContent , innerHtml y outerHTML. La diferencia es que la primera devuelve todo el texto, la segunda devuelve el texto con las etiquetas HTML, y la tercera devuelve el texto con las etiquetas HTML y sus alrededores.

La forma de crear elementos es la siguiente:

Con Document .createElement(TAG) creamos un elemento con el tag indicado (tenemos que ponerlo en mayúsculas y sin "<" y ">" ).
Para añadirle texto, podemos usar el método createTextNode(texto) del elemento recién creado o bien la propiedad innerHtml.
Una vez hecho eso, lo ponemos como hijo del nodo que queramos con el método appendChild(hijo).

Si vamos a añadir muchos elementos, conviene mejor utilizar Document.createDocumentFragment(htmlCode), añadir todos los elementos al fragmento y entonces añadir el fragmento a la página, ya que es más óptimo en recursos, pues de la otra forma destruye todo el DOM cada vez que añades un elemento y lo vuelve a repintar y de esta forma solo lo hace al añadir el fragmento.

Por supuesto, podemos iterar y hacer operaciones con los padres e hijos de cada nodo usando propiedades y métodos como children, firstChild, firstElementChild, nextSibling, nextElementSibling, replaceChild(),etc.

De estos métodos, uno bastante inportante es closest(query), que permite seleccionar el padre más cercano en la jerarquía que cumple el selector CSS de query.

Para añadir eventos, usamos la función addEventListener(evento, función) del elemento al que queramos añadírselo.

Editar Pseudo-elementos

Para ello, en el CSS añadiremos una clase extra en la que le damos estilo al pseudo-elemento. En JS, añadiremos esa clase a la lista de clases del elemento que queramos, lo que modificará el pseudo-elemento debido a la cascada. También se puede reemplazar la clase original por la nueva.

for (elem in array){

La diferencia entre ambos es que in recorre los índices del array, mientras que of recorre los valores.

For each

Existe otra forma de hacer un foreach, y es que los arrays tienen un método llamado forEach que recibe una función como argumento.

Podemos nombrar los bucles simplemente poniendo el nombre seguido de dos puntos. Si tenemos bucles anidados, podemos poner el nombre del array que estamos recorriendo después un break o contiue para que afecte solo a ese bucle.
Un ejemplo de esto sería:

nombre:
for (elem in array){
  code;

}

En Javascript, las funciones se declaran precedidas de la palabra clave function y se ponen los argumentos sin especificar el tipo. Estas se pueden incluso guardar en variables.
También hay lambda functions, que se escriben así:

(parametros)=>{
  code;
}
¡Copiado!

Si sólo tienen una línea de código, además de ejecutarse, se retorna la función que se haya ejecutado.

Los objetos, arrays y funciones se pasan por referencia; el resto de tipos, por valor.

Debug

Podemos printear por consola el estado de una variable con Console.dir(variable).
Otra forma de debuggear es con el método Console.count() y el método Console.countReset(). El primero devuelve cuántas veces se ha llamado y el otro resetea el contador.

Además, se pueden crear grupos para separar uos logs de otros con Console.group(nombre) y Console.groupEnd().

Formateo de strings

let nombre = "Pepe";
let concat = ´¡Hola ${nombre}!´;

Random

Para generar un número aleatorio entre 0 y 1, se utiliza la función Math.Random().
Para hacerlo en un rango, multiplicamos por (max - min) y le sumamos min.
