Javascript

El código Javascript se ejecuta en el navegador del usuario, nunca en el servidor.

Características

  • Interpretado
  • Orientado a Objetos
  • Case sensitive
  • Prefiere camelCase en lugar de snake_case

Algunos de los conceptos de programación son los siguientes:

Vocabulario

Hay que tener cuidado con el tipado dinámico de JS, pues en un if( "string") va a traducir el string a un valor booleano, que será true siempre que no sea un string vacío.

Sin embargo, if( "String" == true) va a traducir los dos a Number, siendo el primero NaN y el segundo 1. Sin embargo, podemos forzar la conversión a boolean con !!.

Variables

La diferencia entre var y let es que var es una variable global, mientras que el scope de let es el bloque en el que lo ejecutamos.

En Javascript, hay una diferencia entre comparar con == y ===.
== compara el valor, mientras que === compara que sean estrictamente iguales, es decir que tanto el valor como el tipo de dato coincidan.

Funciones Comunes

Callbacks y Promesas

Los callbacks son funciones que se llaman dentro de otras funciones.

Las promesas son un objeto que reciben por constructor una función. Son asíncronas y una de las funciones principales para las que están pensadas para responder al recibir algo del servidor.

Esta función a su vez debe tener dos argumentos, que son un callback resolve y otro reject.

En esta función, llamaremos a reject en caso de que queramos devolver un error. Le podemos pasar un mensaje de error.

Por otra parte, llamaremos a resolve cuando la función haya tenido éxito, y podemos pasarle un argumento con lo que queramos devolver.

Tip

Es similar a cuando en C++ devolvemos 0 si no hay ningún error o un código de error si hay algo mal en una función, pero en lugar de eso llamamos a los callbacks resolve o reject.

Una vez se construye el objeto, la función que le hemos pasado se ejecuta y se guarda el estado de la misma.

Si queremos hacer algo, ejecutaremos la función then(callback), que en caso de haber habido un error levantará una excepción con el parámetro que le pasamos a reject (el cual se puede capturar si le definimos el callback catch), mientras que si no ha habido errores, ejecutará el callback resolve.

Async / Await

Las funciones asíncronas son aquellas que permiten esperar a que se completen operaciones que pueden tardar, como una llamada a servidor, bloqueando el "hilo principal".

La diferencia con simplemente usar promesas es que estas actúan nada más llega el recurso, mientras que en una función asíncrona tú puedes hacer que se ejecuten en el orden que quieras, ya que esperarán cuando lo necesites.

Se marcan con async y dentro de ellas, cuando queremos esperar a una de de estas funciones que tardan, lo hacemos poniendo delante de la llamada la palabra clave await, que recoge el resultado devuelto, por lo que si se ha usado con una promesa, recoge lo que devuelva el resolve o el reject.

Las funciones asíncronas siempre que devuelvan algo será una promesa.

Timers

Sirven para ejecutar código tras pasar una determinada cantidad de tiempo en ms.
Es importante saber que consumen bastantes recursos.

Para establecer un timer, usamos la función setTimeout(función, delay).

También podemos quitarlos con la función clearTimeout(temporizador).

También podemos poner timers periódicos, es decir, que se ejecuten cada x ms con la función setInterval(func, intervalo).

Tip

Podemos ejecutar un intervalo un número determinado de veces si lo convinamos con un timer normal.

Para ello, como cada intervalo es regular, calculamos cuánto tiempo pasaría tras ejecutarlo X veces y ejecutamos un timer con ese tiempo cuya función sea eliminar el intervalo, de esta forma:
const intervalo = setInterval(() => document.write("hola"), 2000); setTimeout((() => { clearInterval(intervalo); }, 6000));
¡Copiado!

Para añadir eventos, usamos la función addEventListener(evento, función, [flujo]) del elemento al que queramos añadírselo.

Si usamos una función lambda, podemos pasarle un argumento, que siempre se interpretará de tipo Event.

Los eventos también se pueden eliminar con removeEventListener(evento, función).

Hay que tener en cuenta que esto solo funciona si le pasamos el nombre de una función en lugar de crear una función lambda.

Hay dos tipos de eventos según orden de ejecución (flujo de eventos): los burbuja (que son los que vinen por defecto) y los captura.

Supongamos el evento click. Cuando un contenedor y su hijo tienen ambos el evento, si son burbuja se ejecutará primero el más específico, es decir, el del hijo, y después los de los padres sucesivamente.

En cambio, si a uno o varios de los padres le ponemos el argumento flujo a true, serán captura, lo y se ejecutarán en orden inverso (primero los que sean captura en orden de padres a hijos).

Clases

Para crear clases, la sintaxis es la siguiente:

class nombre extends padre{ constructor(parametros){ super(parametros); code; } this.var1 = valor; function nombreFuncion(){code;} get getAlgo(){code;} set setAlgo(parametros){code;} }

Como podemos ver, dentro de las clases, las propiedades van precedidas siempre de this, y las propias clases se instancian con la palabra clave new.

Por otra parte, conviene declarar los objetos con const en lugar de let para evitar que se modifiquen, ya que de lo contrario podríamos asignarle otro valor, por ejemplo un número, y perderíamos el objeto.

En cuanto a los getters y setters, se declaran con las palabras clave get y set.
A la hora de llamarlos, no se ponen paréntesis. Por ejemplo, para la clase anterior:

let objeto = new nombre(); let algo = objeto.getAlgo; objeto.setAlgo = parametros;

Peticiones HTTP

Son peticiones que hacemos a servidores externos para recibir una información.

Una API sobre la que hacer pruebas es esta.

No guardan información, es decir, no dejan rastro (no van a aparecer en el historial de navegación).

Utilizamos JSON para comunicarnos con el servidor y un JSON SIEMPRE tiene tanto el nombre de las variables como el valor entre comillas, ya que de lo contrario NO se considera JSON y el servidor puede NO interpretar correctamente el objeto.

GET Vs POST

Mientras que los parámetros son visibles en la URL en GET, lo que lo hace menos seguro, en POST son indetectables, pues TAMPOCO se guardan en caché o el historial de búsqueda.

Por otra parte, la longitud de los datos está limitada en GET, pues el máximo es la longitud de una URL.

AJAX

Acrónimo de Asynchronous Javascript And XML.

Normalmente la página se recarga cuando hacemos una petición porque es una operación síncrona, pero AJAX consigue que la operación de hacer una petición sea asíncrona, por lo que no hace falta recargar la página.

Para usar AJAX, hay que instalar un servidor, por ejemplo XAMPP. Desde su panel de control, iniciaremos Apache y MySQL.

Acto seguido, en la carpeta htdocs pondremos la carpeta con nuestro código, ya que es donde el servidor va a leer.

Entonces usaremos la clase XMLHttpRequest para hacer las peticiones.

AJAX no es soportado por todos los navegadores, por lo que si queremos hacer peticiones en navegadores antiguos conviene hacer un if para saber si está definido, y en caso contrario, usar el objeto ActiveXObject en su lugar de esta forma.

let request; if (window.XMLHttpRequest) request = new XMLHttpRequest(); else request = new ActiveXObject("Microsoft.XMLHTTP");
¡Copiado!

Los dos objetos tienen los mismos métodos principales así que no hay problema.

Para enviar peticiones POST tenemos que modificar los headers de esta forma:

request.setRequestHeader("content-type", "application/json;charset=UTF8");
¡Copiado!

Fetch

Es una forma de hacer peticiones basado en promesas, que sustituyó a AJAX.
No nos devuelve los datos en sí, sino que nos los devuelve encapsulados en una promesa.

Una vez que usamos el método fetch(), tenemos que definir el then() de la promesa, y el objeto que recibe su callback tendrá los métodos para obtner los datos. Estos métodos devuelven otra promesa en la que al definir el then() en el objeto que recibe ya tiene los resultados

Principales Funciones y variables

  • json()

Lo ideal es trabajar con Async y await para esperar las respuestas del servidor y que no nos den undefined cuando trabajemos con ello.

Habría que poner el await tanto en la línea que hacemos el fetch como en la que convertimos los datos:

const peticion = async () => { let peticion = await fetch("https://loquesea.es"); let resultado = await peticion.json(); //Hacemos lo que sea }
¡Copiado!

Axios

Es una librería NO nativa para hacer peticiones. Es una alternativa más moderna a Fetch.
Se basa en HttpRequest, pero es más eficiente porque usa promesas.

Para instalar la libreía, nos vamos a su Github.
Bajamos al apartado CDN y copiamos el script ANTES del resto de scripts. (al menos antes de los que lo usen).

Funciona igual que Fetch pero solo hay una promesa, es decir, no hace falta convertirlo, y los datos están en el objeto data del argumento de la promesa. Ejemplo:

axios("informacion.txt").then((res) => { console.log(res.data); });
¡Copiado!

Axios configura automáticamene los headers, y podemos usar los métodos axios.get() y axios.post(url, data) de la misma forma que el método axios() (este hace un get por defecto) para hacer más específicamente un get o un post

Workflow

En Javascript, también existen los foreach, cuya sintaxis es la siguiente:

for (elem in array) { //code }
¡Copiado!
for (elem of array) { //code }
¡Copiado!

La diferencia entre ambos es que in recorre los índices del array, mientras que of recorre los valores.

For each

Existe otra forma de hacer un foreach, y es que los arrays tienen un método llamado forEach que recibe una función como argumento.

Podemos nombrar los bucles simplemente poniendo el nombre seguido de dos puntos. Si tenemos bucles anidados, podemos poner el nombre del array que estamos recorriendo después un break o contiue para que afecte solo a ese bucle.
Un ejemplo de esto sería:

nombre: for (elem in array) { //code; break nombre; }
¡Copiado!

En Javascript, las funciones se declaran precedidas de la palabra clave function y se ponen los argumentos sin especificar el tipo. Estas se pueden incluso guardar en variables.
También hay lambda functions, que se escriben así:

(parametros) => { //code; }
¡Copiado!

Si sólo tienen una línea de código, además de ejecutarse, se retorna la función que se haya ejecutado.

Los objetos, arrays y funciones se pasan por referencia; el resto de tipos, por valor.

En cuanto a los if/else vs los switch/case/default, los primeros son más rápidos.

Keys

En Javascript, para obtener las keys de los objetos, se utiliza el método estático Object. keys(obj), ya que los objetos en sí no tienen accesibles las keys.

Excepciones

Se manejan con los bloques try y catch (error).

Con el objeto error se pueden hacer varias cosas, como usar la palabra clave typeof para imprimir el tipo en consola.

Por otra parte, si usamos un bloque finally, éste se va a ejecutar sí o sí, haya o no errores, incluso saltándose el flujo, es decir, si hay un return antes, va a ejecutarse igual.

Podemos lanzar errores con la palabra reservada throw. Podemos lanzar strings o incluso clases de error que definamos nosotros.

Debug

Podemos printear por consola el estado de una variable con Console.dir(variable).
Otra forma de debuggear es con el método Console.count() y el método Console.countReset(). El primero devuelve cuántas veces se ha llamado y el otro resetea el contador.

Además, se pueden crear grupos para separar uos logs de otros con Console.group(nombre) y Console.groupEnd().

{ duration: 1, //En ms easing: "linear", iterations: 1, //También admite Infinity direction: "reverse" }
